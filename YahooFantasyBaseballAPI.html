Fetching Rosters from the Yahoo Fantasy Baseball API

June 22nd, 2024

Intro

In this notebook, I perform a call on Yahoo's API to fetch the current rosters from a fantasy baseball league and save the data into a csv format. While Yahoo offers the free API to fetch nearly any data that you want, their documentation (link) is incomplete. So, combining what info they do have in their documentation, combined with knowledge of API call standards (link), I played around and got the fetch I was looking for. I plan to build more calls in the future and utilize them as part of a bigger project, but for now I will stick with fetching rosters for a given week, as requested by our league's commissioner for some retrospective analysis.

Import

I will use the Pandas, JSON, and yahoo_oauth packages that can all be found on PyPI.

import pandas as pd

Fetching the Data

Yahoo's oauth package allows you to perform API calls by establishing your sign in via opening the browser and giving you a code. Once you use the code to establish an oauth session, you can perform as many calls as you need. So, if performing the fetches in a notebook, you can establish the session once and not have to re-sign-in while building and tinkering with the rest of the code. Go here to learn how to use the package.

Next, we will do the most challenging part: building the url. As mentioned earlier, there was a lot of playing around necessary to get my desired output. As you can see, I input the game id, league id, and the week and am looking to get the rosters for all teams in return. The game id is a unique number for a given sport for a given season. They have a list that can be found via Yahoo's documentation (link), under the "Game IDs Table", but unfortunately it is outdated. I was able to find my current game ID by opening my league page and searching "gameId" in the network folder of the browser tools.

You can find the league id the same way, or you can find it much easier by looking in the top left corner of your screen on your league page.

Cleaning the Data

As can be seen by the complexity of the previous query and of the resulting snippet, there's a lot of info given in a complicated manner. I will now perform the laborious task of parsing the data to extract what I want. If you are attempting your own fetch with Yahoo's API, I recommend taking the entire json response, saving it into a file and open it up in an IDE. I opened it up in PyCharm, let it reformat the json to be readable, then folded/unfolded the sections from the top in order to understand it all while I built my query for it. In the first query I just get the number of teams. In the second, I loop through the number of teams, then through the number of players, and query the data for each player. I store it all in a pandas dataframe.

In[6]:
num_teams = raw_data['fantasy_content']['league'][0]['num_teams']
In[7]:
df = pd.DataFrame()
for team_num in range(num_teams):
    team_info = raw_data['fantasy_content']['league'][1]['teams'][str(team_num)]['team'][0]
    team_id = team_info[1]['team_id']
    manager = team_info[20]['managers'][0]['manager']['nickname']
    team_data = raw_data['fantasy_content']['league'][1]['teams'][str(team_num)]['team'][1]['roster']['0']['players']
    for player_num in range(team_data['count']):
        player_data = team_data[str(player_num)]['player']
        idx = df.shape[0] if df.shape[0] else 0
        player_df = pd.DataFrame({
            'week': week,
            'league_id': league_id,
            'team_id': team_id,
            'manager': manager,
            'position': [player_data[1]['selected_position'][1]['position']],
            'player_id': [player_data[0][1]['player_id']],
            'first_name': [player_data[0][2]['name']['ascii_first']],
            'last_name': [player_data[0][2]['name']['ascii_last']],
        }, index = [idx])
        df = pd.concat([df, player_df])
